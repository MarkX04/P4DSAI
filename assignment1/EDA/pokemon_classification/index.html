def iqr_outlier_mask(series):
    q1, q3 = series.quantile([0.25, 0.75])
    iqr = q3 - q1
    lower = q1 - 1.5 * iqr
    upper = q3 + 1.5 * iqr
    return (series < lower) | (series > upper)

def file_md5(path_str, chunk_size=8192):
    h = hashlib.md5()
    with open(path_str, "rb") as f:
        while True:
            block = f.read(chunk_size)
            if not block:
                break
            h.update(block)
    return h.hexdigest()

md5_values = [file_md5(fp) for fp in df["file_path"]]

💡 Key Findings

🔴 Extreme Long-Tail Distribution

898 classes for 2,503 images. 520 classes (57.9%) have ≤2 images. The largest class (Venusaur) has just 8 images. Standard train/val/test splits are impractical for most classes.

🟢 Format Consistency

All 2,503 images are .jpg and decoded as RGB. No format conversion or channel handling is needed in the preprocessing pipeline.

🟢 Clean Metadata

Zero missing values across all 11 critical columns. No corrupted or unreadable files detected (0/2,503).

🟡 Aspect Ratio Outliers

While file size and megapixel outlier rates are near zero, aspect ratio outliers reach 3.84%. These images should be padded/resized carefully during preprocessing.

🟡 Cross-Class Duplicate

1 duplicate hash group spanning Appletun ↔ Flapple (cross-class). This indicates label ambiguity/noise for visually similar Pokémon and needs manual review.

🔴 Split Coverage Gaps

659 classes have no validation sample. 520 classes have ≤1 training sample. Consider k-fold cross-validation or few-shot learning approaches for this dataset.

🟢 No Split Leakage

0 duplicate hash groups leak across splits. The current simulation is leakage-free, though group-aware splitting remains recommended for production safety.

🟡 Image Geometry Variation

Resolutions range from 108×110 to 360×400. Average is ~310×315 but varies per class. Standardized resizing is essential for model input.
