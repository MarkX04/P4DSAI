import pandas as pd

def detect_outliers_iqr(df, col):
    q1 = df[col].quantile(0.25)
    q3 = df[col].quantile(0.75)
    iqr = q3 - q1
    lower = q1 - 1.5 * iqr
    upper = q3 + 1.5 * iqr
    mask  = (df[col] < lower) | (df[col] > upper)
    return {
        'feature': col, 'Q1': q1, 'Q3': q3, 'IQR': iqr,
        'lower_bound': lower, 'upper_bound': upper,
        'num_outliers': mask.sum(),
        'pct_outliers': round(mask.mean() * 100, 2)
    }

num_cols = [
    'Credit_Limit', 'Age',
    'BILL_AMT_1', 'BILL_AMT_2', 'BILL_AMT_3',
    'BILL_AMT_4', 'BILL_AMT_5', 'BILL_AMT_6',
    'PAY_AMT_1', 'PAY_AMT_2', 'PAY_AMT_3',
    'PAY_AMT_4', 'PAY_AMT_5', 'PAY_AMT_6'
]

outlier_df = pd.DataFrame([detect_outliers_iqr(df, c) for c in num_cols])
print(outlier_df[['feature', 'num_outliers', 'pct_outliers',
                   'lower_bound', 'upper_bound']].to_string(index=False))

🔢 Cramér’s V — Categorical Association Strength

          

            
          

        

        

          

            Cramér’s V measures the strength of association between each categorical
            or ordinal feature and the binary default target on a [0, 1] scale.
            Unlike Pearson’s r (which requires continuous variables), Cramér’s V works for any nominal or ordinal variable.
          

          

            0.00–0.10 Negligible
            0.10–0.20 Weak
            0.20–0.40 Moderate
            0.40–0.60 Relatively Strong
            0.60+ Strong
          


          

            

              
Cramér’s V vs Default_Payment_Next_Month

              
            

          


          

            Key finding:
            PAY_1 (most recent repayment status) has by far the strongest categorical association
            with default (V ≈ 0.42 — Relatively Strong).
            PAY_2–6 follow with decreasing V, all in the Moderate range (0.25–0.34).
            Demographics (Gender, Education, Marriage) have Negligible V values (< 0.08),
            confirming that payment history dominates over demographics by a wide margin.
          


          

            

              

                
              

              
            

            
import numpy as np
from scipy.stats import chi2_contingency

def cramers_v(col_a, col_b):
    ct = pd.crosstab(col_a, col_b)
    chi2, _, _, _ = chi2_contingency(ct)
    n = ct.sum().sum()
    k = min(ct.shape) - 1
    return float(np.sqrt(chi2 / (n * k))) if k > 0 else 0.0

cat_cols = ['Gender', 'Education', 'Marriage',
            'PAY_1', 'PAY_2', 'PAY_3', 'PAY_4', 'PAY_5', 'PAY_6']
results = {c: cramers_v(df[c], df['Default_Payment_Next_Month']) for c in cat_cols}

📐 Mutual Information — Feature Importance Proxy

          

            
          

        

        

          

            Mutual Information (MI) measures how much knowing feature X reduces uncertainty
            about the target Y, capturing both linear and non-linear dependencies.
            It is a model-agnostic feature importance ranking computed before any classifier is trained.
            Scores are non-negative: 0 = no information about the target.
          


          

            

              
MI Score — All Features vs Default_Payment_Next_Month

              
            

          


          

            Key finding:
            PAY_1–6 occupy the top 6 ranks by MI score, with PAY_1 alone
            contributing ~3× the information of Credit_Limit (rank 7).
            BILL_AMT columns rank lower than expected because the signal is captured more efficiently
            by PAY variables. Demographics rank last. MI confirms and extends the Pearson/Cramér’s V findings.
          


          

            

              

                
              

              
            

            
from sklearn.feature_selection import mutual_info_classif
import numpy as np

feature_cols = ['Credit_Limit', 'Gender', 'Education', 'Marriage', 'Age',
                'PAY_1', 'PAY_2', 'PAY_3', 'PAY_4', 'PAY_5', 'PAY_6',
                'BILL_AMT_1', 'BILL_AMT_2', 'BILL_AMT_3',
                'PAY_AMT_1', 'PAY_AMT_2', 'PAY_AMT_3']

discrete_idx = [feature_cols.index(c) for c in
                ['Gender', 'Education', 'Marriage',
                 'PAY_1', 'PAY_2', 'PAY_3', 'PAY_4', 'PAY_5', 'PAY_6']]

mi = mutual_info_classif(df[feature_cols], df['Default_Payment_Next_Month'],
                         discrete_features=discrete_idx, random_state=42)
mi_series = pd.Series(mi, index=feature_cols).sort_values(ascending=False)
print(mi_series)

💡 Key Insights & Conclusion

          
        

        


          
Dataset & Class Balance

          
    
            
  • 30,000 samples, 23 features, 0 missing values
    • 
            
  • ⚠️ Imbalanced target: No Default 77.88 % vs Default 22.12 % — ratio 3.52 : 1
    • 
            
  • 💡 Avoid accuracy as a metric; use F1-score, AUC-ROC, or Precision-Recall
    • 
          


          
Payment History (PAY_1–PAY_6)

          
    
            
  • PAY_1 is the strongest single predictor — default rate rises from ~13 % (PAY_1 = −2) to ~76 % (PAY_1 = 3)
    • 
            
  • Cramér's V: PAY_1 ≈ 0.42 vs demographics < 0.08 — PAY features dominate by a wide margin
    • 
            
  • Mutual Information: all 6 PAY variables occupy the top 6 ranks before any other feature type
    • 
            
  • 💡 Engineer delay_count = (PAY_k ≥ 1).sum() as a single ordinal risk score (0–6)
    • 
            
  • 💡 Engineer binary has_paid_k = (PAY_AMT_k > 0) for each month as a compact default signal
    • 
          


          
Numerical Features

          
    
            
  • Lower credit limit → higher default risk: ≤ NT$50 K defaults at ~32 % vs ~14 % above NT$270 K
    • 
            
  • BILL_AMT_1–6 are near-collinear (Pearson r ≈ 0.80–0.95) — severe multicollinearity within the group
    • 
            
  • 8–10 % outlier rate in all BILL_AMT and PAY_AMT columns (IQR method)
    • 
            
  • Over-limit clients (util_rate > 100 %) default at ≈ 30 %
    • 
            
  • 💡 For linear models: use only BILL_AMT_1 or apply PCA; tree-based models handle collinearity natively
    • 
            
  • 💡 Engineer util_rate = BILL_AMT_1 / Credit_Limit as a credit utilization feature
    • 
          


          
Categorical & Demographic Features

          
    
            
  • Education: grad school ~19 % default vs high school ~25 % — modest signal
    • 
            
  • Age: clients 20–30 and 60+ have slightly elevated risk; 30–40 group is the safest
    • 
            
  • Gender and Marriage have negligible association with default (Cramér's V < 0.04)
    • 
            
  • 💡 Demographics are best used in combination with payment history, not alone
    • 
          


          

            Overall Conclusion: Payment history (PAY_1–PAY_6) dominates predictive power — confirmed by
            Pearson correlation, Cramér's V, and Mutual Information. Demographics add modest independent signal.
            Recommended modelling steps: handle class imbalance → engineer delay_count & has_paid flags →
            reduce BILL_AMT dimensions via PCA → benchmark Random Forest and XGBoost.
